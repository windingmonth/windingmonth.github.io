Nginx提供的全局变量

都是使用$符号开头的变量，在nginx中大量使用，可以适当背一下。

知道这些变量之后，可以对请求做条件判断。

if ($request_method !~ ^(GET|POST)$ ) { return 405; }
!~ 表示区分大小写的不匹配规则
^ & 这两个符号代表正则的开始和结束，里面就是正则表达式 (GET|POST)

所以该规则只允许用户通过GET或POST方法访问，否则返回405

alias和root的区别

在指定静态文件目录时，通常会使用两个关键字[alias],[root].
两者的区别如下

root对路径的处理：root路径 ＋ location路径
alias对路径的处理：使用alias路径替换location路径

这么说可能有点抽象，来看看例子吧

//等于访问了 /usr/share/nginx/html/static
location ^~ /static {                         
    alias /usr/share/nginx/html/static; # 静态资源路径
}

//也是访问了 /usr/share/nginx/html/static, 只不过它是root/location拼接而来
location ^~ /static {                         
    root /usr/share/nginx/html; # 静态资源路径
}

listence 80 和 listenct [::]:80 的区别

看到过很多类似的配置，他们的区别仅仅是IPv4和IPv6的不同表示而已。

listen 80 default_server;表示监听IP4的80端口以及域名[default_server]
listen [::]:80 default_server;表示监听IP6的80端口以及域名[default_server]

upstream的配置

轮询(默认)

权重(weight)

ip_hash
每个请求按访问ip的hash结果分配，这样每个访客固定访问一个后端服务器，可以解决session的问题。

url_hash
同ip_hash,只不过是用请求的url进行hash

fair(第三方)
按后端服务器的响应时间来分配请求，响应时间短的优先分配。
需要安装nginx-upstream-fair。
